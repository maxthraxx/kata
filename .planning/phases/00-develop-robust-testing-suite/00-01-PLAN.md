  <files>tests/harness/affected.js</files>
  <action>
Create `tests/harness/affected.js` that detects which skills changed based on git diff:

1. Export `getAffectedSkills(baseBranch = 'origin/main')`:
   - Run `git diff --name-only ${baseBranch}...HEAD`
   - Parse changed files to identify affected skills
   - Direct skill changes: `skills/kata-{name}/` -> skill affected
   - Agent changes: `agents/kata-{agent}.md` -> find skills that spawn that agent
   - Return array of skill names (e.g., ['kata-tracking-progress', 'kata-adding-phases'])

2. Export `getAffectedTestFiles(baseBranch = 'origin/main')`:
   - Call getAffectedSkills()
   - Map skill names to test file paths: `kata-{name}` -> `tests/skills/{name}.test.js`
   - Filter to only return files that exist
   - Return array of test file paths

3. Export `getSkillsUsingAgent(agentName)`:
   - Scan all SKILL.md files for agent references
   - Return skills that reference the given agent
   - Cache results for efficiency

Use `node:child_process` execSync and `node:fs` for file operations. Handle cases where git diff returns empty or fails gracefully.
